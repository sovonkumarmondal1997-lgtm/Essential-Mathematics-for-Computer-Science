# Iterative and Recursive Algorithms - Beginner Friendly Notes

These notes explain iterative algorithms, recursive algorithms, and
common looping techniques using simple language, diagrams, real-world
analogies, and step-by-step examples.

------------------------------------------------------------------------

# 1. Iterative Algorithm

## Definition

An **Iterative Algorithm** solves a problem by repeating a set of
instructions using loops such as **for** or **while**.

Instead of calling itself, the algorithm continues until the loop
condition becomes false.

------------------------------------------------------------------------

## Real-Life Analogy

Imagine climbing a staircase.

You take one step.

Then another.

Then another.

You keep repeating until you reach the top.

------------------------------------------------------------------------

## Example 1

Print numbers from 1 to 5.

    1
    2
    3
    4
    5

The algorithm repeats the same action five times.

------------------------------------------------------------------------

## Example 2

Find the sum of numbers from 1 to 4.

    1 + 2 + 3 + 4 = 10

A loop repeatedly adds one number at a time.

------------------------------------------------------------------------

# 2. Recursive Algorithm

## Definition

A **Recursive Algorithm** solves a problem by calling itself on a
**smaller version** of the same problem until it reaches a **base
case**.

------------------------------------------------------------------------

## Real-Life Analogy

Imagine Russian nesting dolls.

Each doll contains a smaller doll.

Eventually you reach the smallest doll.

Then you return outward one doll at a time.

------------------------------------------------------------------------

## Example 1

Factorial

    5!

    =

    5 × 4!

Then

    4!

    =

    4 × 3!

Eventually

    1!

    =

    1

------------------------------------------------------------------------

## Example 2

Merge Sort

Split

    8 3 5 1

↓

    8 3

    5 1

Each half is sorted recursively.

Then merged.

------------------------------------------------------------------------

## Example 3

Quick Sort

Choose a pivot.

Split into smaller groups.

Recursively sort both groups.

------------------------------------------------------------------------

# 3. Current Max Initialization

## Definition

When searching for the largest value,

the algorithm first assumes that the **first element** is the largest.

This is called **Current Max Initialization**.

------------------------------------------------------------------------

## Example 1

List

    [12, 8, 25, 17]

Initially

    Current Max = 12

Now comparisons begin.

------------------------------------------------------------------------

## Example 2

List

    [50, 10, 20]

Initially

    Current Max = 50

Since no larger value exists,

50 remains the maximum.

------------------------------------------------------------------------

## Real-Life Analogy

Suppose the first student enters a tallest-person competition.

Until someone taller arrives,

they remain the tallest.

------------------------------------------------------------------------

# 4. Linear Maximum Scan

## Definition

After choosing the initial maximum,

the algorithm scans every remaining element one by one.

Whenever a larger value is found,

the stored maximum is updated.

------------------------------------------------------------------------

## Example

List

    [12,8,25,17]

Current Max

    12

Compare

    8

    ↓

    No change

Compare

    25

    ↓

    Current Max = 25

Compare

    17

    ↓

    No change

Final answer

    25

------------------------------------------------------------------------

## Another Example

    [4,7,2,10]

Updates

    4

    ↓

    7

    ↓

    10

Maximum

    10

------------------------------------------------------------------------

## Real-Life Analogy

Imagine judging the tallest building in a city.

Every time you find a taller building,

it becomes the new tallest.

------------------------------------------------------------------------

# 5. Loop Variable Increment

## Definition

A loop uses a variable (often called **i**) to track the current
position.

After each iteration,

the variable increases by one.

Eventually,

the loop reaches its stopping condition.

------------------------------------------------------------------------

## Example

    i = 0

    ↓

    i = 1

    ↓

    i = 2

    ↓

    i = 3

The loop moves through every element safely.

------------------------------------------------------------------------

## Example 2

List

    [5,9,2]

Iteration

    i=0

    ↓

    i=1

    ↓

    i=2

Each value is visited exactly once.

------------------------------------------------------------------------

## Real-Life Analogy

Reading pages in a book.

Page 1

↓

Page 2

↓

Page 3

You never skip or repeat pages accidentally.

------------------------------------------------------------------------

# Complete Worked Example

Find the maximum number.

List

    [9,3,12,7,15]

Step 1

Initialize

    Current Max = 9

Step 2

Compare

    3

    ↓

    No change

Step 3

Compare

    12

    ↓

    Current Max = 12

Step 4

Compare

    7

    ↓

    No change

Step 5

Compare

    15

    ↓

    Current Max = 15

Finished.

Maximum

    15

------------------------------------------------------------------------

# Time Complexity

Linear Maximum Scan

    O(n)

Every element is checked exactly once.

------------------------------------------------------------------------

# Iterative vs Recursive Comparison

  Iterative                 Recursive
  ------------------------- -----------------------------------------
  Uses loops                Uses self-calling functions
  Usually less memory       Uses call stack memory
  Easy for repeated tasks   Natural for divide-and-conquer problems
  Example: Linear Scan      Example: Merge Sort, Quick Sort

------------------------------------------------------------------------
