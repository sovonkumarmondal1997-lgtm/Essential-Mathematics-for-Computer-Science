# Recursion and Euclid's Algorithm - Beginner Friendly Notes

These notes explain recursion, the Greatest Common Divisor (GCD), and
Euclid's Algorithm using simple language, diagrams, real-world
analogies, and step-by-step examples.

------------------------------------------------------------------------

# 1. Recursion

## Definition

**Recursion** is a programming technique where a function **calls
itself** to solve a smaller version of the same problem.

Instead of solving the whole problem at once, recursion keeps reducing
it until it reaches a simple case that can be solved directly.

------------------------------------------------------------------------

## Real-Life Analogy

Imagine a set of Russian nesting dolls.

Open the largest doll.

Inside is a smaller doll.

Open it.

Inside is another smaller doll.

Eventually you reach the smallest doll.

Then you close them one by one in reverse order.

This is recursion.

------------------------------------------------------------------------

## Example 1: Countdown

Goal:

Print

    3
    2
    1

Recursive thinking:

    Print 3

    ↓

    Ask the same function to print 2

    ↓

    Ask the same function to print 1

    ↓

    Stop

------------------------------------------------------------------------

## Example 2: Factorial

    5!

    =

    5 × 4!

Then

    4!

    =

    4 × 3!

Eventually

    1!

    =

    1

------------------------------------------------------------------------

# 2. Execution Layering (Stack / Onion Analogy)

## Definition

Every recursive call creates a **new execution layer**.

Each layer remembers its own local variables.

When the deepest call finishes, the layers return one by one.

------------------------------------------------------------------------

## Onion Analogy

Imagine peeling an onion.

Each peel is another recursive call.

After reaching the center,

you put the layers back in reverse order.

------------------------------------------------------------------------

## Stack Diagram

    main()

    ↓

    f(3)

    ↓

    f(2)

    ↓

    f(1)

    ↓

    Base Case

Returning

    Base Case

    ↑

    f(1)

    ↑

    f(2)

    ↑

    f(3)

    ↑

    main()

------------------------------------------------------------------------

## Real-Life Analogy

Imagine climbing down a ladder.

You cannot jump back to the top.

You must climb upward one step at a time.

------------------------------------------------------------------------

# 3. Greatest Common Divisor (GCD)

## Definition

The **Greatest Common Divisor (GCD)** is the largest positive integer
that divides two numbers exactly without leaving a remainder.

------------------------------------------------------------------------

## Example 1

Numbers

    12

    18

Factors of 12

    1 2 3 4 6 12

Factors of 18

    1 2 3 6 9 18

Largest common factor

    6

GCD = 6

------------------------------------------------------------------------

## Example 2

Numbers

    20

    30

Common factors

    1

    2

    5

    10

Largest

    10

------------------------------------------------------------------------

# 4. Euclid's Algorithm

## Definition

Euclid's Algorithm efficiently finds the GCD by repeatedly replacing the
larger number with the remainder of division.

Rule

    GCD(a,b)

    =

    GCD(b,a mod b)

Continue until

    b = 0

Then

    a

is the GCD.

------------------------------------------------------------------------

## Example

Find

    GCD(48,18)

Step 1

    48 mod 18 = 12

    ↓

    GCD(18,12)

Step 2

    18 mod 12 = 6

    ↓

    GCD(12,6)

Step 3

    12 mod 6 = 0

    ↓

    GCD(6,0)

Answer

    6

------------------------------------------------------------------------

## Real-Life Analogy

Imagine cutting two ropes into equal pieces.

You repeatedly cut the larger rope using the length of the smaller rope
until no extra piece remains.

The remaining length is the largest possible equal piece.

------------------------------------------------------------------------

# 5. Terminating Gate (Base Case)

## Definition

A **Base Case** is the stopping condition of recursion.

Without it,

recursion would continue forever.

------------------------------------------------------------------------

## Example 1

Factorial

    Factorial(1)

    ↓

    Return 1

------------------------------------------------------------------------

## Example 2

Euclid

    If b == 0

    Return a

This safely stops recursion.

------------------------------------------------------------------------

## Everyday Analogy

Driving to school.

You stop when you arrive.

Otherwise,

you would drive forever.

------------------------------------------------------------------------

# 6. Iterative vs Recursive Implementation

## Definition

There are two common ways to solve many problems.

### Iterative

Uses loops.

Variables are updated repeatedly.

### Recursive

Uses functions that call themselves.

------------------------------------------------------------------------

## Example

Find GCD

### Iterative

    WHILE b != 0

    temp = b

    b = a mod b

    a = temp

------------------------------------------------------------------------

### Recursive

    GCD(a,b)

    IF b == 0

    Return a

    Return GCD(b,a mod b)

------------------------------------------------------------------------

## Real-Life Analogy

Suppose you need to climb ten stairs.

Iterative

Walk one stair at a time.

Recursive

Ask someone to climb the remaining stairs after each step until no
stairs remain.

------------------------------------------------------------------------

# Complete Worked Example

Find

    GCD(36,24)

Step 1

    36 mod 24 = 12

    ↓

    GCD(24,12)

Step 2

    24 mod 12 = 0

    ↓

    GCD(12,0)

Base Case reached.

Answer

    12

------------------------------------------------------------------------

## Recursive Call Stack

    GCD(36,24)

    ↓

    GCD(24,12)

    ↓

    GCD(12,0)

    ↓

    Return 12

    ↑

    Return 12

    ↑

    Return 12

------------------------------------------------------------------------

# Time Complexity

Euclid's Algorithm

    O(log n)

Very efficient even for very large numbers.

------------------------------------------------------------------------

# Iterative vs Recursive Comparison

  -----------------------------------------------------------------------
  Iterative                           Recursive
  ----------------------------------- -----------------------------------
  Uses loops                          Uses self-calling functions

  Usually less memory                 Uses call stack memory

  Easier for simple repetition        Easier for naturally recursive
                                      problems

  No function-call overhead           Function-call overhead exists
  -----------------------------------------------------------------------
