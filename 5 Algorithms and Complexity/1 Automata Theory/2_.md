# Finite Automata (FA) - Simple Notes

## 1. Finite Automaton (FA)

### Definition

A **Finite Automaton (FA)** is a simple mathematical machine that reads
an input string **one symbol at a time** and decides whether to
**accept** or **reject** the string.

It has **limited memory** because it only remembers its **current
state**.

### Think of it Like...

Imagine a security guard checking an ID card.

-   Looks at one character at a time
-   Follows a fixed set of rules
-   Ends by saying:
    -   "Access Granted" (Accept)
    -   "Access Denied" (Reject)

The guard never remembers the entire ID card---only the current checking
step.

------------------------------------------------------------------------

### Example 1

Alphabet:

    Σ = {0,1}

Accept only strings ending in **1**

Accepted:

    1
    01
    101
    111

Rejected:

    0
    10
    1100

------------------------------------------------------------------------

### Example 2

Accept only the word

    cat

Accepted:

    cat

Rejected:

    ca
    cats
    bat
    cut

------------------------------------------------------------------------

## 2. Read-Only Tape

### Definition

The **Read-Only Tape** stores the input string.

The automaton can:

-   Read symbols
-   Move forward

It **cannot change** any symbol.

------------------------------------------------------------------------

### Example 1

Input:

    hello

Tape:

    +---+---+---+---+---+
    | h | e | l | l | o |
    +---+---+---+---+---+

The machine reads each letter but never changes them.

------------------------------------------------------------------------

### Example 2

Binary input:

    10101

Tape:

    +---+---+---+---+---+
    |1|0|1|0|1|
    +---+---+---+---+---+

Still read-only.

------------------------------------------------------------------------

## 3. Tape Head

### Definition

The **Tape Head** reads **one symbol at a time** and moves to the next
cell.

------------------------------------------------------------------------

### Example 1

Input:

    abc

Step 1

    ^
    a b c

Reads:

    a

Step 2

      ^
    a b c

Reads:

    b

Step 3

        ^
    a b c

Reads:

    c

------------------------------------------------------------------------

### Example 2

Binary:

    1101

The tape head reads

    1 → 1 → 0 → 1

------------------------------------------------------------------------

## 4. States (Q)

### Definition

A **state** represents the machine's current condition.

States are usually drawn as circles.

Think of states as checkpoints.

------------------------------------------------------------------------

### Example 1

Door Lock

    Locked
    Unlocked

Current state tells whether the door is locked.

------------------------------------------------------------------------

### Example 2

Traffic Light

    Red
    Yellow
    Green

Each color is a state.

------------------------------------------------------------------------

### Example 3

Password Checker

    Start
    Correct First Letter
    Correct Second Letter
    Accepted

------------------------------------------------------------------------

## 5. Transitions

### Definition

Transitions are arrows connecting states.

They tell the machine:

> "If you read this symbol, move here."

------------------------------------------------------------------------

### Example 1

    (q0) --a--> (q1)

Meaning:

If currently in q0 and read "a", move to q1.

------------------------------------------------------------------------

### Example 2

    (q1) --b--> (q2)

Reading b moves to q2.

------------------------------------------------------------------------

### Example 3

Traffic light

    Red
    ↓

    Green
    ↓

    Yellow
    ↓

    Red

Each arrow is a transition.

------------------------------------------------------------------------

## 6. Initial State (q₀)

### Definition

The **Initial State** is where the automaton always starts.

Diagram:

    → (q0)

Incoming arrow from nowhere indicates the start.

------------------------------------------------------------------------

### Example 1

Word checker

    Start

    ↓

    q0

Machine always begins at q0.

------------------------------------------------------------------------

### Example 2

ATM

    Insert Card

    ↓

    PIN Verification

    ↓

    Transactions

"Insert Card" acts like the initial state.

------------------------------------------------------------------------

## 7. Accepting States (F)

### Definition

Accepting states are special states drawn with **double circles**.

If all input has been read and the machine is in one of these states,
the string is accepted.

Otherwise rejected.

------------------------------------------------------------------------

### Example 1

    (q0) --a--> ((q1))

Double circle:

    ((q1))

is accepting.

Input:

    a

Accepted.

Input:

    aa

Depends on later transitions.

------------------------------------------------------------------------

### Example 2

Checking strings ending with 1

Ending state:

    ((EndsWith1))

Strings ending in 1 finish here.

Accepted.

------------------------------------------------------------------------

## 8. Five-Tuple Formal Definition

Every finite automaton is written as

    (Q, Σ, δ, q₀, F)

Meaning:

  Symbol   Meaning
  -------- ---------------------
  Q        Set of states
  Σ        Alphabet
  δ        Transition function
  q₀       Initial state
  F        Accepting states

------------------------------------------------------------------------

### Example

    Q = {q0,q1}

    Σ = {0,1}

    q0 = Start

    F = {q1}

Transition rules:

    δ(q0,1)=q1

    δ(q1,0)=q0

This completely defines the automaton.

------------------------------------------------------------------------

## 9. Transition Function (δ)

### Definition

The transition function tells the automaton:

> "Given the current state and current input symbol, where should I go
> next?"

Mathematically:

    δ : Q × Σ → Q

------------------------------------------------------------------------

### Example 1

States:

    q0
    q1

Alphabet:

    0
    1

Rules:

    δ(q0,0)=q0

    δ(q0,1)=q1

    δ(q1,0)=q0

    δ(q1,1)=q1

Meaning:

If in q0 and read 1, move to q1.

------------------------------------------------------------------------

### Example 2

Traffic Light

    δ(Red,Timer)=Green

    δ(Green,Timer)=Yellow

    δ(Yellow,Timer)=Red

------------------------------------------------------------------------

## 10. Transition Table

### Definition

Instead of writing many δ rules, we can organize them into a table.

Rows = Current State

Columns = Input Symbol

Cell = Next State

------------------------------------------------------------------------

### Example

Alphabet

    Σ={0,1}

  Current State   0    1
  --------------- ---- ----
  q0              q0   q1
  q1              q0   q1

Interpretation:

Current state:

    q0

Read:

    1

Move to

    q1

------------------------------------------------------------------------

# Complete Working Example

Machine accepts strings ending in **1**

States

    Q={q0,q1}

Alphabet

    Σ={0,1}

Start

    q0

Accepting

    F={q1}

Transition Table

  State   0    1
  ------- ---- ----
  q0      q0   q1
  q1      q0   q1

Example Input

    1011

Processing

    Start → q0

    Read 1 → q1

    Read 0 → q0

    Read 1 → q1

    Read 1 → q1

Final state:

    q1

Since q1 is accepting,

    Accepted

------------------------------------------------------------------------

# Quick Summary

  Concept               Simple Meaning
  --------------------- -----------------------------------------
  Finite Automaton      Machine that accepts or rejects strings
  Read-Only Tape        Stores the input and cannot be modified
  Tape Head             Reads one symbol at a time
  States                Current condition of the machine
  Transitions           Rules for moving between states
  Initial State         Starting state
  Accepting State       Final state that accepts input
  Five-Tuple            (Q, Σ, δ, q₀, F) defines the automaton
  Transition Function   Decides next state
  Transition Table      Organized form of transition rules
