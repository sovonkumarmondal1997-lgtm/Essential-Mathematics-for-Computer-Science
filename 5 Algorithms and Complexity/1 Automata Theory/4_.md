# Automaton Simulator and Finite State Machines (FSM) - Beginner Friendly Notes

## 1. Automaton Simulator

### Definition

An **Automaton Simulator** is a software tool that lets you **draw**,
**run**, and **test** finite automata such as **DFAs** and **NFAs**.

Instead of solving everything on paper, the simulator shows how the
automaton moves from one state to another while reading an input string.

Think of it as a **virtual laboratory** for finite automata.

------------------------------------------------------------------------

### What can an Automaton Simulator do?

-   Create states
-   Connect states with transitions
-   Mark accepting states
-   Enter input strings
-   Show each computation step
-   Tell whether a string is accepted or rejected
-   Test many strings automatically

------------------------------------------------------------------------

### Example 1

Suppose you want to accept strings ending with **1**.

You draw:

    → q0 ----1----> ((q1))
      ^             |
      |             |
      +------0------+

Input:

    101

The simulator highlights:

    q0

    ↓

    q1

    ↓

    q0

    ↓

    q1

Finally:

    Accepted

------------------------------------------------------------------------

### Example 2

You design a machine accepting only

    cat

Test strings:

    cat      ✓

    cats     ✗

    bat      ✗

    ca       ✗

The simulator immediately tells which strings are correct.

------------------------------------------------------------------------

## 2. Finite State Machine (FSM / FSA)

### Definition

A **Finite State Machine (FSM)** or **Finite State Automaton (FSA)** is
a mathematical model consisting of:

-   States
-   Transitions
-   Start state
-   Accept state(s)

It processes one symbol at a time to decide whether an input belongs to
a language.

------------------------------------------------------------------------

### Real-Life Example 1: Vending Machine

States:

    No Coin

    ↓

    Coin Inserted

    ↓

    Product Dispensed

Each button press or coin insertion changes the state.

------------------------------------------------------------------------

### Real-Life Example 2: Elevator

States:

    Floor 1

    Floor 2

    Floor 3

Pressing buttons changes the elevator's current state.

------------------------------------------------------------------------

### Computer Science Example

Password checker

    Start

    ↓

    Correct First Letter

    ↓

    Correct Second Letter

    ↓

    Accepted

------------------------------------------------------------------------

## 3. DFA and NFA

### DFA (Deterministic Finite Automaton)

A DFA has **exactly one possible next state** for every input symbol.

Think:

"There is only one road to follow."

------------------------------------------------------------------------

### Example

    State q0

    Read 0

    ↓

    q1

There is no second choice.

------------------------------------------------------------------------

### NFA (Non-Deterministic Finite Automaton)

An NFA may have:

-   Multiple possible next states
-   No transition
-   ε-transitions (moves without reading input)

Think:

"You may choose one of several roads."

------------------------------------------------------------------------

### Example

Reading

    a

from q0 may go to

    q1

    or

    q2

------------------------------------------------------------------------

### Simple Comparison

  DFA                 NFA
  ------------------- -----------------------
  One possible path   Many possible paths
  No ε-transitions    ε-transitions allowed
  Easier to execute   Easier to design

------------------------------------------------------------------------

## 4. Start State

### Definition

The **Start State** is where every computation begins.

In most simulators it is automatically created.

Diagram

    → q0

The incoming arrow means:

"Start here."

------------------------------------------------------------------------

### Example 1

Input:

    101

Before reading anything:

    Current State

    q0

------------------------------------------------------------------------

### Example 2

Even if the string is empty

    ε

The computation still starts at

    q0

------------------------------------------------------------------------

## 5. Transitions and Labels

### Definition

Transitions are arrows connecting states.

Labels tell which symbol causes the move.

------------------------------------------------------------------------

### Example

    (q0) ----a----> (q1)

Meaning:

If currently in q0 and read **a**,

move to q1.

------------------------------------------------------------------------

### Example 2

    (q1) ----b----> (q2)

Reading b moves to q2.

------------------------------------------------------------------------

### Example 3

Multiple labels

    (q0) ----0,1----> (q1)

Reading either 0 or 1 follows the same path.

------------------------------------------------------------------------

## 6. Self-Loop

### Definition

A self-loop is a transition that begins and ends at the same state.

The machine stays in the same state while reading certain symbols.

------------------------------------------------------------------------

### Example 1

            a
          ↺
    ((q0))

Every a keeps the machine in q0.

------------------------------------------------------------------------

### Example 2

Accept strings containing only zeros.

          0
        ↺
    ((q0))

Input:

    000000

The machine never leaves q0.

Accepted.

------------------------------------------------------------------------

### Example 3

Waiting room analogy

You stay in the same room until something different happens.

------------------------------------------------------------------------

## 7. Accept State Selection

### Definition

A state becomes an accepting state only after you explicitly mark it in
the simulator.

Usually this appears as:

-   Double circle
-   Checkbox
-   "Final State" option

------------------------------------------------------------------------

### Example 1

Normal state

    (q1)

Accept state

    ((q1))

------------------------------------------------------------------------

### Example 2

Suppose q2 is the goal.

Only strings finishing in q2 are accepted.

If the machine ends in q1,

Rejected.

------------------------------------------------------------------------

## 8. Automated Test Cases

### Definition

Most simulators provide boxes where you enter:

-   Strings that should be accepted
-   Strings that should be rejected

The simulator checks every test automatically.

------------------------------------------------------------------------

### Example

Language:

Strings ending with **1**

Accepted Tests

    1

    01

    101

    111

Rejected Tests

    0

    10

    1100

    100

The simulator reports

    ✓ Passed

    or

    ✗ Failed

for every test.

------------------------------------------------------------------------

### Example 2

Language:

Even number of 1s

Accepted

    ε

    11

    1010

    1100

Rejected

    1

    111

    101

Automatic testing quickly verifies whether the automaton is correct.
