# Finite Automata (FA) - Advanced Concepts in Simple Terms

## 1. Graph Representation

### Definition

A **Graph Representation** is a visual diagram of a finite automaton.

-   Circles represent **states**.
-   Directed arrows represent **transitions**.
-   Labels on arrows indicate the input symbol causing the move.

### Example 1: Strings ending with `1`

            1
    → (q0) -----> ((q1))
       ^            |
       |0           |1
       |            |
       +------------+
            0

-   `q0` is the start state.
-   `q1` is the accepting state.
-   Reading symbols moves between states.

### Example 2: Traffic Light

    (Red) ---> (Green)
       ^            |
       |            |
    (Yellow) <------+

Each circle is a state, and arrows show the next state.

------------------------------------------------------------------------

## 2. Loop (Self-Transition)

### Definition

A **Loop** (or **Self-Transition**) is an arrow that starts and ends at
the same state.

The machine stays in the same state after reading a particular symbol.

### Example 1

          a
       ↺
    (q0)

Reading `a` keeps the automaton in `q0`.

### Example 2

Accept strings containing only `0`s.

           0
        ↺
    ((q0))

Every `0` keeps the machine in `q0`.

### Example 3

Waiting room analogy:

If no new instruction arrives, you remain in the waiting room.

------------------------------------------------------------------------

## 3. Acceptance Condition

### Definition

A string is accepted **only if**:

1.  Every input symbol has been read.
2.  The automaton stops in an accepting state.

Both conditions must be true.

### Example 1

Machine:

    → q0 --a--> ((q1))

Input:

    a

End at `q1`.

Accepted.

### Example 2

Input:

    aa

Suppose reading the second `a` returns to `q0`.

Final state:

    q0

Rejected.

Even though the machine visited `q1`, it did not finish there.

------------------------------------------------------------------------

## 4. Intermediate Passing

### Definition

Passing through an accepting state **during the computation** does
**not** mean the string is accepted.

Acceptance is checked **only after the last symbol is read**.

### Example 1

    → q0 --a--> ((q1)) --b--> q2

Input:

    ab

Path:

    q0 → q1 → q2

Although `q1` is accepting, the machine ends in `q2`.

Rejected.

### Example 2

Think of traveling.

Passing through your destination city does not count if your journey
ends elsewhere.

------------------------------------------------------------------------

## 5. State Characterization (Parity Tracking)

### Definition

States can represent whether the count of a symbol seen so far is:

-   Even
-   Odd

This is called **Parity Tracking**.

### Example 1: Count of `1`s

State meanings:

    qEven = even number of 1s

    qOdd = odd number of 1s

Input:

    1011

Tracking:

    Start (Even)

    1 → Odd

    0 → Odd

    1 → Even

    1 → Odd

Ends in Odd.

### Example 2: Count of `a`

Input:

    abaa

Occurrences of `a`:

    0 → Even

    1 → Odd

    1 → Odd

    2 → Even

    3 → Odd

The automaton changes state whenever another `a` appears.

------------------------------------------------------------------------

## 6. Initial State Condition (q₀)

### Definition

Before reading any symbol:

-   Zero `a`s have been seen.
-   Zero `b`s have been seen.
-   Zero is an even number.

Therefore the initial state represents **even counts for all symbols**.

### Example 1

Alphabet:

    Σ={0,1}

Before reading input:

    0 zeros

    0 ones

Both are even.

### Example 2

Input:

    ε

No symbols read.

Still:

    Even zeros

    Even ones

------------------------------------------------------------------------

## 7. Transition Mapping

### Definition

Transition Mapping means deciding how each input symbol changes the
state.

Ask:

"What changes after reading this symbol?"

### Example 1

Track parity of `1`s.

    Current: Even

    Read 1

    ↓

    Odd

Another `1`:

    Odd

    ↓

    Even

### Example 2

Reading `0`

    Even

    ↓

    Even

Because the number of `1`s does not change.

------------------------------------------------------------------------

## 8. Design Completeness Rule

### Definition

For every state,

there must be a transition for **every symbol** in the alphabet.

Otherwise the automaton is incomplete.

### Example 1

Alphabet

    Σ={0,1}

States

    q0

    q1

Correct design

  State   0    1
  ------- ---- ----
  q0      q0   q1
  q1      q0   q1

Every cell has a transition.

### Example 2

Incomplete

  State   0    1
  ------- ---- ----
  q0      q0   q1
  q1      q0   

Missing transition.

Machine does not know what to do.

------------------------------------------------------------------------

## 9. Computation Tracing (Execution)

### Definition

Computation tracing means following the machine step by step while it
processes the input.

### Example 1

Machine accepts strings ending in `1`.

Transition Table

  State   0    1
  ------- ---- ----
  q0      q0   q1
  q1      q0   q1

Input

    101

Execution

    Start

    ↓

    q0

    Read 1

    ↓

    q1

    Read 0

    ↓

    q0

    Read 1

    ↓

    q1

Input finished.

Final state:

    q1

Accepted.

### Example 2

Input

    1100

Execution

    Start → q0

    1 → q1

    1 → q1

    0 → q0

    0 → q0

Ends at `q0`.

Rejected.

------------------------------------------------------------------------

# Complete Example

Goal:

Accept strings containing an even number of `1`s.

States

    qEven

    qOdd

Start

    qEven

Accepting

    qEven

Transition Table

  State   0       1
  ------- ------- -------
  qEven   qEven   qOdd
  qOdd    qOdd    qEven

Input

    10110

Trace

    Start

    ↓

    qEven

    1 → qOdd

    0 → qOdd

    1 → qEven

    1 → qOdd

    0 → qOdd

Final state

    qOdd

Rejected because the number of `1`s is odd.

------------------------------------------------------------------------

# Quick Summary

  -----------------------------------------------------------------------
  Concept                   Simple Meaning
  ------------------------- ---------------------------------------------
  Graph Representation      Diagram of states and transitions

  Loop                      Arrow returning to the same state

  Acceptance Condition      Read entire input and finish in an accepting
                            state

  Intermediate Passing      Visiting an accepting state in the middle
                            does not count

  State Characterization    States represent information such as even/odd
                            counts

  Initial State Condition   Before reading input, all counts are zero
                            (even)

  Transition Mapping        Decide the next state for each input symbol

  Design Completeness Rule  Every state must have transitions for every
                            input symbol

  Computation Tracing       Follow the automaton step by step through the
                            input
  -----------------------------------------------------------------------
