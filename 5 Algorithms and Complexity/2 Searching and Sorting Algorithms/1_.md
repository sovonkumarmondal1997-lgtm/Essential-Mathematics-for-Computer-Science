# Algorithms - Beginner Friendly Notes

These notes explain the fundamental concepts of algorithms in simple
language with real-world analogies, step-by-step examples, and revision
summaries.

------------------------------------------------------------------------

# 1. Algorithm

## Definition

An **algorithm** is a **finite, ordered, unambiguous set of executable
steps** used to solve a specific problem by converting **inputs into
outputs**.

Simply put:

> An algorithm is a recipe or a set of instructions that tells a
> computer exactly how to solve a problem.

------------------------------------------------------------------------

## Real-Life Analogy

Imagine making a cup of tea.

Steps:

1.  Boil water.
2.  Add tea leaves.
3.  Add sugar.
4.  Add milk.
5.  Pour into a cup.

These steps form an algorithm because they are:

-   Ordered
-   Clear
-   Executable
-   Finish after a few steps

------------------------------------------------------------------------

## Example 1: Add Two Numbers

### Problem

Add two numbers.

### Input

    5
    7

### Algorithm

1.  Read first number.
2.  Read second number.
3.  Add them.
4.  Display the result.

### Output

    12

------------------------------------------------------------------------

## Example 2: Find Maximum Number

Input

    10
    25

Steps

1.  Compare both numbers.
2.  Choose the larger one.
3.  Print it.

Output

    25

------------------------------------------------------------------------

# 2. Input, Output, and Problem

Every algorithm has three basic parts.

## Problem

What needs to be solved.

## Input

The data given to solve the problem.

## Output

The final answer produced.

------------------------------------------------------------------------

## Example 1

Problem:

Find the area of a rectangle.

Input:

    Length = 5

    Width = 4

Output

    20

------------------------------------------------------------------------

## Example 2

Problem

Check whether a number is even.

Input

    8

Output

    Even

    Problem

    ↓

    Input

# 3. Algorithm Life Cycle

Creating a program involves several stages.

## Step 1

Understand the problem.

Example:

"Find the largest number."

------------------------------------------------------------------------

## Step 2

Design the algorithm.

Example

    Compare both numbers.

    Return the larger one.

------------------------------------------------------------------------

## Step 3

Check correctness.

Try sample inputs.

------------------------------------------------------------------------

## Step 4

Analyze the algorithm.

Estimate:

-   Running time
-   Memory usage

------------------------------------------------------------------------

## Step 5

Implement the algorithm.

Write code.

Example:

Python

    if a>b:
        print(a)

------------------------------------------------------------------------

## Step 6

Test the program.

Inputs

    5 8

    10 2

    3 3

Verify the answers.

------------------------------------------------------------------------

## Life Cycle Diagram

    Problem

    ↓

    Design

    ↓

    Correctness

    ↓

    Analysis

    ↓

    Coding

    ↓

    Testing

------------------------------------------------------------------------

# 4. Algorithmic Analysis (Time and Space Complexity)

Algorithms are compared by:

-   Time
-   Memory

------------------------------------------------------------------------

## Time Complexity

Measures how long an algorithm takes.

Example

Search in a list.

Small list

    10 numbers

Fast.

Large list

    10 million numbers

Slower.

------------------------------------------------------------------------

## Space Complexity

Measures how much memory is used.

Example

Store

    100 numbers

Needs little memory.

Store

    10 million numbers

Needs much more memory.

------------------------------------------------------------------------

## Everyday Analogy

Travel.

Time

    How long does the trip take?

Space

    How much luggage do you carry?

------------------------------------------------------------------------

# 5. Ordered Criteria

## Definition

Every step must appear in the correct order.

The computer must always know what comes next.

------------------------------------------------------------------------

## Correct Example

    1. Read a

    2. Read b

    3. Add

    4. Print

------------------------------------------------------------------------

## Incorrect Example

    Print answer

    ↓

    Read numbers

    ↓

    Add

The order is wrong.

------------------------------------------------------------------------

## Cooking Analogy

Bake the cake

before

mixing ingredients.

Impossible.

------------------------------------------------------------------------

# 6. Unambiguous Criteria

## Definition

Each instruction must have exactly one meaning.

No vague wording.

------------------------------------------------------------------------

## Good Example

    Multiply x by 5.

Very clear.

------------------------------------------------------------------------

## Bad Example

    Multiply x by a large number.

How large?

Nobody knows.

------------------------------------------------------------------------

## Another Bad Example

    Sort quickly.

How?

Ascending?

Descending?

The instruction is unclear.

------------------------------------------------------------------------

# 7. Executable Criteria

## Definition

Every instruction must actually be possible.

------------------------------------------------------------------------

## Good Example

    Add two numbers.

Possible.

------------------------------------------------------------------------

## Bad Example

    Divide by zero.

Impossible.

------------------------------------------------------------------------

## Another Bad Example

    Read a value that does not exist.

The computer cannot perform it.

------------------------------------------------------------------------

## Real-Life Analogy

Instruction

    Jump to the Moon.

Impossible.

Therefore,

not executable.

------------------------------------------------------------------------

# 8. Terminating Process (Finiteness)

## Definition

An algorithm must always stop after a finite number of steps.

It must never run forever.

------------------------------------------------------------------------

## Good Example

    Print numbers

    1

    2

    3

    4

    5

    Stop

Ends after five steps.

------------------------------------------------------------------------

## Bad Example

    Print Hello forever.

    Hello

    Hello

    Hello

    Hello

    ...

Never stops.

Not a valid algorithm.

------------------------------------------------------------------------

## Everyday Analogy

Suppose someone asks:

"Keep walking forever."

You never reach your destination.

That is not useful.

------------------------------------------------------------------------

# Complete Example

Problem

Find the sum of three numbers.

Input

    5

    10

    15

Algorithm

    Step 1

    Read first number.

    ↓

    Step 2

    Read second number.

    ↓

    Step 3

    Read third number.

    ↓

    Step 4

    Add all numbers.

    ↓

    Step 5

    ↓

This algorithm is:

✓ Ordered

✓ Unambiguous

✓ Executable

✓ Finite
