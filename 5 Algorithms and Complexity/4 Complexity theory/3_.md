# Asymptotic Analysis - Beginner Friendly Notes

These notes explain asymptotic behavior and algorithm growth using
simple language, real-world analogies, and step-by-step examples.

------------------------------------------------------------------------

# 1. Asymptotic Behavior

## Definition

**Asymptotic Behavior** studies how an algorithm's running time grows as
the input size (**n**) becomes **very large**.

Instead of asking:

"How many seconds does it take?"

we ask:

"How does the running time grow when the input becomes bigger and
bigger?"

This helps us understand whether an algorithm **scales well**.

## Example 1

Algorithm A

    n

Algorithm B

    n²

For

    n = 10

A → 10 operations

B → 100 operations

For

    n = 1000

A → 1000 operations

B → 1,000,000 operations

Algorithm A grows much more slowly.

## Example 2

Searching:

-   Linear Search → O(n)
-   Binary Search → O(log n)

As the list grows larger, Binary Search stays much faster.

## Real-Life Analogy

Walking vs. climbing stairs.

Walking 10 meters is easy.

Walking 10 kilometers takes much longer.

Asymptotic behavior studies how effort grows as distance becomes very
large.

------------------------------------------------------------------------

# 2. Dominant Term Isolation

## Definition

When analyzing complexity, we keep only the **fastest-growing term**.

We ignore:

-   constants
-   lower-order terms

because they matter less for very large inputs.

## Example 1

    5n² + 7n + 10

Dominant term

    n²

Complexity

    O(n²)

## Example 2

    3n log n + 8n + 100

Dominant term

    n log n

Complexity

    O(n log n)

## Real-Life Analogy

Imagine climbing a mountain.

The steepest part determines how difficult the climb feels.

Small flat sections barely matter.

------------------------------------------------------------------------

# 3. Complexity Class Hierarchy

## Definition

Different algorithms grow at different speeds.

Slower growth means better scalability.

From fastest algorithms (slowest growth) to slowest algorithms (fastest
growth):

  Complexity   Growth
  ------------ --------------
  O(1)         Constant
  O(log n)     Logarithmic
  O(n)         Linear
  O(n log n)   Linearithmic
  O(n²)        Quadratic
  O(n³)        Cubic
  O(2ⁿ)        Exponential

## Example

Suppose

    n = 1000

Approximate work

-   O(1) → 1
-   O(log n) → about 10
-   O(n) → 1000
-   O(n log n) → about 10000
-   O(n²) → 1,000,000
-   O(n³) → 1,000,000,000

Notice how quickly quadratic and cubic algorithms become expensive.

## Real-Life Analogy

Transportation speeds:

-   Elevator → O(1)
-   Bicycle → O(log n)
-   Walking → O(n)
-   Crawling → O(n²)

The slower the movement, the longer it takes to reach the destination.

------------------------------------------------------------------------

# 4. Asymptotic Function Sorting

## Definition

Sometimes several mathematical functions are given.

We rank them from **best (slowest growth)** to **worst (fastest
growth)** by looking only at their dominant terms.

## Example 1

Functions

    7

    3 log n

    4n

    2n²

    5n³

Sorted

    7

    ↓

    3 log n

    ↓

    4n

    ↓

    2n²

    ↓

    5n³

Equivalent complexity

    O(1)

    ↓

    O(log n)

    ↓

    O(n)

    ↓

    O(n²)

    ↓

    O(n³)

## Example 2

Functions

    6n² + 5

    9n

    8n log n

    2ⁿ

Dominant terms

    n²

    n

    n log n

    2ⁿ

Sorted

    O(n)

    ↓

    O(n log n)

    ↓

    O(n²)

    ↓

    O(2ⁿ)

## Real-Life Analogy

Imagine runners in a marathon.

The runner who slows down the least finishes first.

Algorithms are ranked similarly based on how quickly their work grows.

# Worked Example

Simplify

    8n³ + 2n² + 100n + 500

Step 1

Largest-growing term

    8n³

Step 2

Ignore constant coefficient

    n³

Final complexity

    O(n³)

# Quick Summary

-   Asymptotic Behavior: Studies growth as n becomes very large.
-   Dominant Term Isolation: Keep only the fastest-growing term.
-   Complexity Class Hierarchy: Rank algorithms by growth rate.
-   Asymptotic Function Sorting: Compare functions using their dominant
    terms.
