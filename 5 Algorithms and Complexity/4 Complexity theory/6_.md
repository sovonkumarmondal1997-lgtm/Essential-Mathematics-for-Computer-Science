# Master Theorem - Beginner Friendly Notes

These notes explain the Master Theorem and related divide-and-conquer
concepts using simple language, analogies, and examples.

------------------------------------------------------------------------

# 1. Master Theorem

The **Master Theorem** is a shortcut for finding the time complexity of
many divide-and-conquer recurrences.

General form:

    T(n)=aT(n/b)+O(n^d)

Instead of expanding the recurrence manually, compare the recursive work
with the non-recursive work.

### Example

Merge Sort:

    T(n)=2T(n/2)+n

Master Theorem immediately gives

    O(n log n)

------------------------------------------------------------------------

# 2. Subproblem and Splitting Parameters (a and b)

-   **a** = number of recursive subproblems.
-   **b** = factor by which the input size is divided.

### Example 1

    T(n)=2T(n/2)+n

a = 2

b = 2

### Example 2

    T(n)=4T(n/2)+n

a = 4

b = 2

Analogy: Cut a pizza into smaller pieces. **a** is how many pieces you
make, **b** tells how much smaller each piece is.

------------------------------------------------------------------------

# 3. Local Work Exponent (d)

The value **d** comes from the work done outside recursion.

### Example

    T(n)=2T(n/2)+n²

Local work is

    n²

Therefore

    d=2

Another example:

    T(n)=T(n/2)+1

Constant work:

    d=0

------------------------------------------------------------------------

# 4. Leaf-Dominated Condition

Condition:

    d < log_b(a)

The recursive calls grow faster than the local work.

Result:

    O(n^(log_b a))

### Example

    T(n)=2T(n/2)+1

a=2

b=2

d=0

    log₂2=1

Since

    0<1

Complexity:

    O(n)

------------------------------------------------------------------------

# 5. Balanced-Work Condition

Condition:

    d = log_b(a)

Every recursion level performs about the same amount of work.

Result:

    O(n^d log n)

### Example

Merge Sort

    T(n)=2T(n/2)+n

a=2

b=2

d=1

    log₂2=1

Equal values give

    O(n log n)

------------------------------------------------------------------------

# 6. Root-Dominated Condition

Condition:

    d > log_b(a)

The non-recursive work dominates.

Result:

    O(n^d)

### Example

    T(n)=T(n/2)+n

a=1

b=2

d=1

    log₂1=0

Since

    1>0

Complexity:

    O(n)

------------------------------------------------------------------------

# 7. Binary Search Master Theorem Proof

Binary Search recurrence:

    T(n)=T(n/2)+1

Parameters:

a=1

b=2

d=0

    log₂1=0

Balanced condition:

    d=log_b(a)

Therefore

    O(log n)

because

    n^0 log n = log n

------------------------------------------------------------------------

# 8. Recursive Maximum Finder

Split an array into two halves.

Find the maximum of each half recursively.

Compare the two maxima.

### Example

    [7,2,9,5]

Split:

    [7,2]

    [9,5]

Maximums:

    7

    9

Compare:

    max(7,9)=9

------------------------------------------------------------------------

# 9. Leaf-Dominated Linear Recurrence

Recurrence:

    T(n)=2T(n/2)+1

Parameters:

a=2

b=2

d=0

Leaf work dominates.

Complexity:

    O(n)

------------------------------------------------------------------------

# 10. Root-Dominated Linear Recurrence

Recurrence:

    T(n)=T(n/2)+n

Parameters:

a=1

b=2

d=1

Root work dominates.

Complexity:

    O(n)

------------------------------------------------------------------------

# Master Theorem Summary

  Case             Condition       Complexity
  ---------------- --------------- -----------------
  Leaf Dominated   d \< log_b(a)   O(n\^(log_b a))
  Balanced         d = log_b(a)    O(n\^d log n)
  Root Dominated   d \> log_b(a)   O(n\^d)

------------------------------------------------------------------------

# Worked Examples

  Recurrence       Result
  ---------------- ------------
  T(n)=T(n/2)+1    O(log n)
  T(n)=2T(n/2)+1   O(n)
  T(n)=2T(n/2)+n   O(n log n)
  T(n)=T(n/2)+n    O(n)

------------------------------------------------------------------------

# Quick Summary

-   Master Theorem quickly solves divide-and-conquer recurrences.
-   a = number of recursive calls.
-   b = input reduction factor.
-   d = local work exponent.
-   Compare d with log_b(a) to choose the correct case.
