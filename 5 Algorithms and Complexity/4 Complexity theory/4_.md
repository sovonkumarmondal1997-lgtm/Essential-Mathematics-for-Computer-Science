# Big O Notation and Complexity - Beginner Friendly Notes

## 1. Big O Notation

Big O describes the upper bound of an algorithm's growth as input size
increases.

Example: - Linear Search → O(n) - Binary Search → O(log n)

It measures growth, not exact seconds.

------------------------------------------------------------------------

## 2. Witnesses (c and k)

To prove f(n)=O(g(n)), find constants c and k so that

f(n) ≤ c·g(n)

for every n ≥ k.

Example:

f(n)=3n+2

For n≥2:

3n+2 ≤ 4n

Therefore: c=4, k=2.

------------------------------------------------------------------------

## 3. Upper Bound Growth Property

A function grows slower than or equal to a scaled version of another
function after some point.

Example:

5n+10 ≤ 6n (for n≥10)

Therefore:

5n+10 = O(n)

------------------------------------------------------------------------

## 4. Big O Algebraic Verification

Replace smaller terms with larger ones.

Example:

4n²+4n

Since 4n≤4n² for n≥1,

4n²+4n≤8n²

Therefore O(n²).

------------------------------------------------------------------------

## 5. Logarithmic Exponent Simplification

Rule:

log(a\^b)=b log(a)

Example:

log(2\^n)=n log2

Since log2 is constant,

log(2\^n)=O(n).

------------------------------------------------------------------------

## 6. Big O Linear Verification

Example:

2n+7

For n≥7,

2n+7≤3n

Witnesses:

c=3

k=7

Therefore O(n).

------------------------------------------------------------------------

## 7. Dominant Exponential Growth Proof

Example:

5n and 2\^n

At n=5:

5n=25

2\^n=32

As n grows,

2\^n grows much faster.

Therefore:

5n=O(2\^n).

------------------------------------------------------------------------

## 8. Comparative Complexity Dominance

Example:

n = O(n²)

log n = O(n)

But

n² is not O(n).

------------------------------------------------------------------------

## 9. Linear Minimum Search Complexity

Finding the minimum requires checking every element.

Example:

\[8,3,9,2,6\]

Current minimum changes:

8→3→2

Complexity:

O(n).

------------------------------------------------------------------------

## 10. Selection Sort Time Complexity

Selection Sort repeatedly performs a linear minimum search.

Passes:

5,4,3,2,1 comparisons

Overall:

O(n²).

------------------------------------------------------------------------

## 11. Constant Time Operations

Examples:

-   Variable initialization
-   Swap two values
-   Read one array element

Each takes O(1).

------------------------------------------------------------------------

## 12. Uniform Case Complexity Balance

Selection Sort always scans the remaining list completely.

Best: O(n²)

Average: O(n²)

Worst: O(n²)

------------------------------------------------------------------------

## Worked Example

6n²+2n+10

For n≥1:

2n≤2n²

10≤10n²

Therefore:

6n²+2n+10≤18n²

Witnesses:

c=18

k=1

So:

6n²+2n+10=O(n²)

------------------------------------------------------------------------

## Complexity Summary

  Complexity   Example
  ------------ -----------------------------
  O(1)         Swap
  O(log n)     Binary Search
  O(n)         Linear Search
  O(n log n)   Merge Sort
  O(n²)        Bubble Sort, Selection Sort
  O(2ⁿ)        Naive Fibonacci

------------------------------------------------------------------------

## Quick Summary

-   Big O measures upper-bound growth.
-   Witnesses (c,k) prove Big O.
-   Ignore constants and lower-order terms.
-   Exponential functions eventually dominate linear ones.
-   Minimum search is O(n).
-   Selection Sort is O(n²) in all cases.
-   O(1) operations take constant time.
