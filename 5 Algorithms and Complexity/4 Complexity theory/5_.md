# Recurrence Relations and Recursion Tree Analysis - Beginner Friendly Notes

These notes explain recurrence relations and common methods for
analyzing recursive algorithms using simple language, diagrams,
analogies, and step-by-step examples.

------------------------------------------------------------------------

# 1. Recurrence Relation

## Definition

A **Recurrence Relation** is a mathematical equation that describes the
running time of a recursive algorithm in terms of **smaller inputs**.

Instead of giving the answer directly, it explains how today's problem
depends on smaller versions of the same problem.

### Example 1

Merge Sort

    T(n) = 2T(n/2) + n

Meaning:

-   Solve two smaller problems of size n/2.
-   Spend n extra work to merge them.

### Example 2

Binary Search

    T(n) = T(n/2) + 1

Meaning:

-   Search only one half.
-   Do one comparison.

### Real-Life Analogy

Imagine stacking boxes.

To move one large box, you first split it into two smaller boxes, move
them, then finish the remaining work.

------------------------------------------------------------------------

# 2. Substitution Method (Guess and Induction)

## Definition

The **Substitution Method** solves a recurrence by:

1.  Guessing the answer.
2.  Proving the guess using mathematical induction.

### Example

Recurrence

    T(n)=T(n/2)+1

Guess

    T(n)=O(log n)

Then use induction to prove the guess is always true.

### Another Example

    T(n)=2T(n/2)+n

Guess

    O(n log n)

Verify mathematically.

------------------------------------------------------------------------

# 3. Recursion Tree Method

## Definition

A **Recursion Tree** draws every recursive call as a tree.

Each level contributes some work.

Add the work from all levels to find the total running time.

### Example

    T(n)=2T(n/2)+n

Tree

            n

         n/2   n/2

      n/4 n/4 n/4 n/4

Each level contributes

    n

Total work depends on the number of levels.

------------------------------------------------------------------------

# 4. Logarithmic Tree Height Rule

## Definition

If the input keeps being divided by 2, the tree height becomes

    log n

because

    n

    ↓

    n/2

    ↓

    n/4

    ↓

    ...

    ↓

    1

### Example

    n=16

Levels

    16

    8

    4

    2

    1

Height

    log₂16 = 4

### Another Example

    32

    16

    8

    4

    2

    1

Height

    5

# 5. Recursive Array Summation

## Definition

Instead of adding every element one by one, divide the array into two
halves.

Recursively find each half's sum.

Then add the two results.

### Example

Array

    [2,4,6,8]

Split

    [2,4]

    [6,8]

Split again

    [2]

    [4]

    [6]

    [8]

Return

    2+4=6

    6+8=14

    6+14=20

Final sum

    20

### Analogy

Imagine two friends counting coins in two piles.

Each counts one pile.

Then combine the totals.

------------------------------------------------------------------------

# 6. Geometric Series Tree Expansion

## Definition

Consider

    T(n)=2T(n/2)+1

Every level doubles the number of nodes.

Each node performs constant work.

The total work forms a **geometric series**.

### Example

Level 0

    1 node

Level 1

    2 nodes

Level 2

    4 nodes

Level 3

    8 nodes

Total work

    1+2+4+8+...

This geometric series grows to

    O(n)

because the total number of nodes is proportional to n.

### Real-Life Analogy

A family tree where every parent has two children.

Each generation doubles in size.

# Common Recurrence Examples

  Recurrence       Complexity
  ---------------- ------------
  T(n)=T(n-1)+1    O(n)
  T(n)=T(n/2)+1    O(log n)
  T(n)=2T(n/2)+1   O(n)
  T(n)=2T(n/2)+n   O(n log n)

------------------------------------------------------------------------

# Quick Summary

-   Recurrence Relation: Expresses a recursive algorithm using smaller
    inputs.
-   Substitution Method: Guess the complexity and prove it.
-   Recursion Tree: Add work across all tree levels.
-   Logarithmic Tree Height: Repeated halving gives log n levels.
-   Recursive Array Summation: Split, sum each half, combine.
-   Geometric Series Tree Expansion: Doubling nodes creates a geometric
    series that often simplifies to O(n).
